supergit · SVG showcase

Ten SVG techniques that are underused as live UI primitives, each wired to something supergit actually shows — usage warnings, agent activity, worktree topology, diffs. Every demo is hand-rolled SVG/CSS, no libraries. Drag the sliders, hit the buttons.

01 Usage rings — stroke-dasharray + pathLength

Two concentric circles: a faint track and a colored arc. Setting pathLength="100" means the dash math is just percentages — no 2πr arithmetic. A second, dimmer arc underneath shows projected usage behind actual. The stroke color flips through your existing warning-kind enum (ok → pace → warn → hot).

The trick: pathLength="100" normalizes the path so stroke-dasharray: 64 100 is literally "64%". Animate stroke-dashoffset (or the dasharray) for the fill-up tween. Rounded ends via stroke-linecap:round.

In supergit: the weekly / session usage warnings you ship as toasts (pace, 90% weekly, 95% session) become a glanceable ring on each agent chip in AgentUsageChip.svelte. The ghost arc = projected weekly, the solid arc = actual — so "on track to blow the cap" is visible before the toast fires.

02 Sparklines — one <polyline>, no chart lib

A row of N numbers → a single points string. Layer a translucent area <path> underneath for body, and a "now" <circle> at the last point. Cheap enough to render one per session row.

line only · gradient stroke
bars · per-commit churn
In supergit: a usage-over-time sparkline next to each agent in ProcessList.svelte / the session dock, or per-repo commit churn in RepoRecentSummary.svelte. No dependency, scales to any DPI, themeable via currentColor.

03 Branch graph — Bézier <path> + <marker> arrowheads

Git topology is a node-edge graph. Curved <path> edges with a reusable <marker> arrowhead draw the Fork-style commit lanes without a canvas. (Visualizing topology — not a rebase UI. Stays clear of "open in Fork" territory.)

main · feature · worktree-x
In supergit: a mini-graph in GitHistory.svelte showing how each worktree has diverged from main — ahead/behind as lane length, merge points as joins. Worktree-first dashboards live and die on "how far has this drifted," and a glyph beats a "+3 −1" text badge.

04 <textPath> — labels that ride the curve

Text flows along an arbitrary path instead of sitting in a flat box. Branch names follow their lane; a status label can arc around a ring.

branch name follows its lane
arced label around a status ring
In supergit: long branch names (feature/usage-warning-toasts) curving along the branch-graph lanes from demo 3, or a "SYNCING…" label arcing around the usage ring while a remote pull is in flight.

05 <animateMotion> — a pulse traveling down a branch

Move a dot along a path declaratively. A glowing pulse runs down the branch lane when an agent commits or a push lands — directional liveness you can't fake with a CSS fade.

data flows main → worktree

Caveat: SMIL (<animateMotion>) is fine in WebKit/Electron but has a rocky standards history. For load-bearing animation in the native app, CSS offset-path is the safer equivalent. This demo uses SMIL for brevity.

In supergit: SSE change events (sse-change-kinds.ts) trigger a pulse along the relevant worktree lane — a commit, a push, an incoming remote sync each get a directional blip instead of a silent row re-render.

06 Self-drawing strokes — animate stroke-dashoffset

Set the dash to the full path length and animate the offset from full → 0: the path appears to draw itself. The canonical "operation succeeded" checkmark, or a spinner that traces a ring.

success check
failed op
working… trace
In supergit: the result state for any daemon op that resolves — share-session, remote-sync upload, repair-session. LoadingSpinner.svelte / SleepIndicationAnimation.svelte already animate; a drawn check on completion closes the loop visually.

07 Filters — feGaussianBlur goo, feTurbulence haze, feMorphology glow

The deep end. Goo: blur + a high-contrast alpha matrix makes nearby blobs fuse. Haze: animated feTurbulence + feDisplacementMap is a "thinking" shimmer. Glow: feMorphology dilates an outline for focus rings.

gooey merge · agents converging
heat haze · agent is thinking
focus glow · selected worktree

Perf caveat: animated feTurbulence across 50 rows will cook a laptop fan. Reserve filters for hover/active/selected state, or pre-bake. feDropShadow is the cheap one and themes better than CSS box-shadow on irregular shapes.

In supergit: the haze marks an actively-streaming agent (a calmer SleepIndicationAnimation sibling); the goo fuses two agent dots when sessions share a worktree; the morphology glow is the selection ring for the focused worktree row (session-focus-store.ts).

08 <clipPath> & <mask> — wipes and faded edges

A clip shows only what's inside a shape (hard edge). A mask uses luminance/alpha, so a gradient mask fades content to transparent. Great for a diff "wipe" reveal and for fading the ends of an overflowing strip to hint "there's more, scroll."

before / after diff wipe
edge fade · "scroll for more"
In supergit: the wipe is a swipe-to-compare for DiffViewer.svelte (before/after on top of each other, drag the seam). The gradient fade is the soft edge on the terminal-output preview strips in ShellPreview.svelte / ChatPreview.svelte so truncation reads as "fades out," not "abruptly cut."

09 <pattern> — texture for stale / conflicted state

Tile a shape into a fill. Diagonal hatch for "stale" worktrees, a hazard-stripe for "conflicted," a dot grid for "no activity." Scales cleanly and inherits currentColor, so one pattern themes across light/dark.

In supergit: the hatch overlays a worktree row (worktree-row.css) that's behind its remote or hasn't been touched in days; the hazard stripe flags a row with merge conflicts — a texture cue that survives even when the status color is colorblind-ambiguous.

10 Animated gradients — moving stops & skeleton shimmer

Gradient <stop>s can animate. A skeleton-loader sweep is one moving highlight stop; a usage bar's green→amber→red is three stops you reposition as fill grows, so the color escalation is the gradient itself, not a JS color switch.

usage bar · escalating gradient
skeleton · loading session row
In supergit: the escalating bar is the linear cousin of the demo-1 ring for the context-size chip (context-tokens.ts, --ctx-warn / --ctx-hot); the shimmer is the loading state for session/repo rows before the daemon responds (LoadingOverlay.svelte).

11 currentColor + CSS vars — one icon, every theme

Inline SVG inherits color and can read var(--…) directly. One icon set auto-themes with accent / status / hover — no duplicate light/dark assets, no recoloring in JS. Hover the swatches; the same three glyphs recolor live.

Rule of thumb: inline SVG > <img src="x.svg"> for anything you want to style, animate, or theme — an <img> is an opaque box.

In supergit: directly supports the design-token consolidation work — icons.ts / app-icons.ts / AgentIcon.svelte glyphs reference currentColor and var(--brand) / var(--status-*), so a single token edit re-skins every icon at once.
